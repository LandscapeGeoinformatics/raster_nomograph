Abstract

Recommending optimal widths for riparian buffer zones is still a challenging task, in particular when considering the spatial variability of the landscape. Empirical-based approaches include assessment of terrain, soil types, land use and vegetation, and are often realised in computationally expensive hydrological simulation. However, trade-offs have to be made between spatial resolution and areal extent. Another elegant empirical-based approach are nomographs, where via triangulation of a specific slope length, terrain slope and soil type recommended buffer width can be easily calculated. Mander and Kuusemets (1998) already developed such a nomograph for Estonian catchments in 1998, yet, a computational use case has not been explored.

We implemented the nomograph as a basic GIS algorithm to calculate the specific slopelength in order to retrieve the recommended buffer width. As inputs we use slope (5m DEM, Landboard 2019) and soil types (EstSoil-EH, 2021). We synthesisized the specific slopelength via a weighted average of flow length, slope length and LS factor, as can be easily calculated by any GIS. In the last step we digitised the defined nomogrpah angles and provide a lookup function. We applied this algorithm and calculated buffer strip widths for the whole of Estonia, over an area of approx 43,000 km2. We evaluated the uncertainty of the results as well the algorithms sensitivity to input weights.

This exercise directly informs policy making by being able to more specifically decide and explain variations of buffer zone widths along streams, rivers and all water protections zones in general based on a scientific location-specific analysis.

Introduction

riparian buffer zones serve as filter for surface and (very) shallow runoff from the landscape. In particular in regards to nutrient fluxes from agricultural fields, riparian buffer strips can mitigate degradation of river water quality. While widely acknowledging the effectiveness, much debate is still ongoing to agree on recommendable widths. This is partically due to either typically very localised studies on efficiency on one side with difficulties in upscaling to catchment or even regional scale. On the other side karger scale GIS-based assessments on pollution sensitivty draw generalised conclusions and don't take local landscape and flow patterns into account.

To bridge this gap we are combining specific localised knowledge on buffer zone efficiency and scale it up to larger regions while embracing local variations such as flow patterns, variations in soil and terrain. In order to overcome computational burdens of nutrient flux simulation models we use empiracally-based nomograph function of Mander and Kusemets, which is validated on several Estonian catchments and captures the dynamics of nutrient fluxes on local scale for effective and specific recommendation of buffer strip widths. We take this nomograph and implement this as a raster-based GIS application. This pixel-based softare implementation can now be scaled at various solutions. GIS raster algebra calculations are computationally much less heavy than numerical model simulations, and have the advantage of being fully spatially distributed at the same time.

In order to make the computations for all of Estonia, we divided the processing area into 22 zones. The processing zones which have overlap with each other to ensure the spatial dependencies are maintained which are necessary for calculating flow path length and flow accumulation. For each zone we added a 5 km buffer in order to capture hydrological continuity across the zone boundaries. This way we make sure we don't create artificial interruptions along the flow direction paths. At the very end of the data processing the zones are clipped back to their original extent. The joined zones don't show any artifacts along their borders which confirms the effectiveness of the zoning method.

Implementation as a QGIS processing algorithm:

The core of the digital nomogram is the relationship of the angles on the nomogram. These are implemented as Python NumPy trigonometric functions. With our data preprocessing we have now prepared per pixel inputs for the key variables:

  1. the specific slope length (based on the three input factors)
  2. the slope - angle in degrees
  3. the soil type, i.e. texture class

The tool loads the five input raster layers, which are provided in GeoTIFF format. It is crucial that the input raster are aligned in their origin point and resolution. The algorithm then applies the og transformation, the normalisation, the weighing of input layers, calculates the specific slope length and then calculates the recommendable buffer width based on the nomogram trigonmetric relationship function. As final output the algorithm writes out two raster layers with the same alignment and resolution as the input layers for the recommnedable buffer size as well as the calculated specific slopelength for reference.

Synthesising recommendable buffer strip widths along streams, rivers and water protection zones:

To better inform policy makers and catchment management we capture recommendable buffer zone widths along river and streams into a vector-based dataset extending the official river network data.

In GIS, geomorphology, catchment delineation it is a well-known fact that real-world rivers and streams do not always align perfectly with DEM derived flow lines. These may be artifacts in the digital elevation model or modifications of the river beds that are not (yey) reflected in the DEM.

We segmentize/subdivide the stream network into 5 meters long segments. Smaller resolution is not necessary as the raster resolution is also 5m. Then we define one-sided buffers along the river line segments of 50 meters each and perform zonal statistics for each stream segment buffer against the recommendable buffer widths raster dataset and take the median value to capture an acceptable local buffer width value, even if the stream bed defined by the vector layer does not exactly align with the hydrological dynamics derived from the DEM.

Discussion points:

Why median? We needed to capture the dynamic of each local zone. The maximum value would likely be too harsh, and mean might be too influenced by outliers or a strong skew. Median captures robustly the local recommendable buffer width.

To perform the sensitivity analysis of weighing factors contributing to “Specific Slope Length f(m)” we apply a global sensitivity analysis called "Extended FAST" based on Saltelli et al. (1999). This is besides the Sobol sensitivity test the only algorithm available that is taking parameter interaction into account.

Eq: N = ( 1 + 4 x M**2) x ( 1 + ( k + 2 ) x d ) ) x k
Eq: N = ( 1 + 4 x M**2) x ( 1 + ( k + 2 ) x d ) ) x k

with N = needed parameter iterations, M= inference factor (default M=4) and d= frequency step (default d=2) and k = 3 as the number of parameters of the model: N = 579

Discussion points:

For flow accumulation of the GRASS GIS algorithm for all 22 processing zones the largest values in Estonia reach ca. 21000. To better capture the topographic effects for water flow behaviour in the landscape we are integrating flow length and ls factor values. These values are much smaller and not directly comparable within the nomograph function. Thus, we normalised all 3 values into 0-100, weight and average them and then need to scale up again. This reduces extreme values from numerical raster-based flow accumulation and increases attention to local hill slopes close to streams. We call this the "Nomograph Catchment Co-Efficient". Based on the value of 21000 for the max flow accumulation and the normalisation to 100 for the data integration with flow length and ls factor we are testing with the upscaling value of about 200 (21000 / 100 = 210).
